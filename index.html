Architecture

Clean separation of concerns. Every layer has one job.

🗺️

Routes

Map HTTP methods and URLs to controller functions. Zero business logic.

authRoutes.js productRoutes.js categoryRoutes.js

All business logic lives here. Reads requests, queries models, sends responses.

JWT verification, global error handling, and request validation run before controllers.

What it does

Every feature built with real-world production patterns.

01

Full CRUD

Create, read, update and delete products and categories with consistent response structure across all endpoints.

02

Advanced Filtering

Filter by category, price range, stock level, tags, and active status. Combine filters freely via query params.

03

Pagination & Search

Cursor-style pagination with configurable limits. Full-text search via MongoDB regex on name and description.

04

JWT Authentication

Stateless auth via signed tokens. Role-based access control separates public reads from admin writes.

05

Error Middleware

Global handler normalizes every error — Mongoose, JWT, validation, duplicates — into consistent JSON responses.

06

Swagger Docs

Auto-generated interactive docs at /api/docs. Every endpoint, parameter, and response is documented.

📦 StockPilot — Inventory Management API

Node.js Express MongoDB v1.0.0 MIT License

A production-grade REST API for inventory and product management. Designed with clean architecture, JWT authentication, advanced filtering, pagination, and interactive Swagger documentation.

🚀 Features

  • Full CRUD for products and categories
  • JWT authentication with role-based access control (admin/user)
  • Advanced filtering: price range, stock, tags, search, category
  • Pagination and sorting on all list endpoints
  • Logical delete (soft delete) preserving data integrity
  • Global error handling middleware with standardized responses
  • Input validation via express-validator
  • Interactive API docs at /api/docs

⚙️ Installation

# Clone the repo
git clone https://github.com/YOUR_USERNAME/stockpilot.git

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env

# Start dev server
npm run dev
